# Security Research Notes

**Status:** Active threat and control notebook
**Normative specification:** docs/09_SECURITY_GOVERNANCE.md

## Purpose

Capture new threats, abuse cases, control alternatives, incident lessons, and external security review findings.

## Threat-note template

- Date
- Asset
- Threat actor
- Entry point
- Preconditions
- Attack path
- Impact
- Existing controls
- Control gap
- Proposed mitigation
- Verification test
- Residual risk
- Architecture decision

## Priority research

- Passkey and recovery design
- Policy language/compilation
- Audit checkpoint signing
- Prompt-injection evaluation
- Parser/computer sandbox
- Plugin trust/signatures
- Data residency
- Secrets Provider
- Free-model/provider privacy
- Computer-control abuse

## Constraint

Security findings containing exploit details or secrets require restricted handling and must not be copied into normal prompts or logs.
